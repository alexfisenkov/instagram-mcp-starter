# Три метода: обзор и как выбирать

Аналитику Instagram нельзя достать целиком из одного источника. Каждый метод закрывает свой слой данных. Ниже — что откуда берётся и в каком порядке подключать.

## Почему методов три

| Слой данных | Где лежит | Каким методом достаётся |
|-------------|-----------|-------------------------|
| Профиль, посты, охваты, показы, базовые инсайты, комментарии | Meta Graph API | **1 — Официальный API** |
| Весь контент профиля, медиафайлы (посты/Reels/карусели) для скачивания и базы, поля, которых нет в API | Веб-версия Instagram (ты залогинен) | **2 — Браузер** |
| Удержание по секундам, кривая просмотров, момент отвала аудитории, детальная аудитория Reels/Stories | **Только приложение на телефоне** | **3 — Телефон** |

Ключевая мысль: **самая ценная статистика для разбора сценариев — удержание и точки отвала — есть только в мобильном приложении** (метод 3). Официальный API её не отдаёт. Веб отдаёт больше, чем API, но тоже не всё. Поэтому для «полного ключа» нужны все три.

## Порядок подключения

```
Метод 1 (официальный API)   ← начни здесь: безопасно, стабильно, база для отчётов
      │
      ▼
Метод 2 (браузер)           ← добавь, когда нужен архив контента и скачивание
      │
      ▼
Метод 3 (телефон)           ← добавь, когда нужна глубина: удержание, отвалы, сценарии
```

Каждый следующий метод не заменяет предыдущий, а добавляет слой. Можно остановиться на любом.

## Сравнение

| | 1 · API | 2 · Браузер | 3 · Телефон |
|---|---------|-------------|-------------|
| Установка | одна команда | + браузерный тулинг | + Appium/adb (iOS нужен Mac) |
| Стабильность | высокая | средняя (Instagram меняет вёрстку) | средняя (зависит от телефона/ОС) |
| Риск для аккаунта | нет | низкий (чтение/сбор, без действий) | средний (чтение своего телефона) |
| Уникальные данные | инсайты по API | архив + скачивание (своё и чужое публичное) | **удержание, отвалы, аудитория своих Reels** |
| Автоматизируемость | отличная | хорошая | ручная/полуручная |

## Плюс метод 4 — разведка по конкурентам

[04-research.md](04-research.md) — не отдельный источник, а связка методов 1–3, направленная на **чужие публичные аккаунты**: собрать общедоступную статистику, скачать доступный контент, расшифровать и разобрать его, проанализировать подписи и комментарии → понять, почему у других заходит, и применить у себя. Удержание/детальную аудиторию чужого аккаунта достать нельзя (это приватно у владельца) — по конкурентам доступны публичные метрики, контент и его разбор.

## Общие правила для всех методов

1. **Полное управление — своим аккаунтом; чтение и сбор — по доступному.** Свой Instagram ведёшь как хочешь; чужое читаешь и собираешь в объёме **общедоступного** (публичные посты, метрики, комментарии). Приватное, к чему нет доступа, — не трогаем.
2. **Не действуй «ботом» по чужим.** Лайки, комментарии, подписки, DM, публикации в чужую сторону, массовые автодействия — не для этого: это спам и бан. Чтение и анализ — сколько нужно.
3. **Не сохраняй секреты как данные.** Куки, токены, session-строки браузера, UDID/серийники телефона — не выгружать в отчёты и не коммитить. В базу идут ссылки, метрики, медиа и скриншоты — не «слепки» сессии.
4. **Уважай лимиты и чужой контент.** Быстрые массовые запросы Instagram замечает — читай спокойно, с паузами. Скачанное чужое — для твоего анализа, не выдавай за своё и не перезаливай.
