# Метод 1 — Официальный Meta Graph API

**Начни отсюда.** Самый безопасный и стабильный способ: официальное API Meta, только чтение, 16 инструментов. Даёт профиль, посты, охваты, показы, базовые инсайты и комментарии. Это база для регулярных отчётов.

## Что даёт

- Информация об аккаунте: имя, username, подписчики, число постов.
- Список постов/Reels с датами, подписями, ссылками, лайками, комментариями.
- Инсайты аккаунта (охват, показы, просмотры профиля) и постов (охват, показы, сохранения, взаимодействия).
- Топ контента по вовлечённости, комментарии и ответы.
- Произвольный read-only GET к разрешённым путям Graph API.

Полный список: [../docs/TOOLS.md](../docs/TOOLS.md).

## Чего НЕ даёт (и зачем нужны методы 2–3)

- Удержание Reels по секундам и точку отвала аудитории → **метод 3 (телефон)**.
- Скачивание медиафайлов и полный архив контента для базы → **метод 2 (браузер)**.
- Часть детальной аудитории и метрик, которых нет в текущих scopes.

## Установка и подключение

Это ровно то, что ставит основной установщик репозитория:

```bash
# macOS / Linux
bash <(curl -fsSL https://raw.githubusercontent.com/alexfisenkov/instagram-mcp-starter/main/install.sh)
```

```powershell
# Windows
irm https://raw.githubusercontent.com/alexfisenkov/instagram-mcp-starter/main/install.ps1 | iex
```

Дальше:

1. **Ключи Meta-приложения** (App ID / App Secret): [../docs/01-meta-app.md](../docs/01-meta-app.md).
2. **Авторизация** (OAuth, получение long-lived токена): [../docs/02-oauth.md](../docs/02-oauth.md).
3. Проверка: попроси Claude выполнить `meta_auth_status`, затем `meta_get_account_info`.

Проще всего поручить всё это агенту — он идёт по [../AGENT.md](../AGENT.md), этапы 0–5.

## Обслуживание

- Токен живёт ~60 дней. Раз в месяц говори агенту «обнови токен Instagram» (`meta_refresh_token`). Доктор предупреждает, когда осталось ≤10 дней.
- Диагностика: `node ~/.instagram-mcp/app/doctor.mjs ~/.instagram-mcp/run.sh`.

Проблемы: [../docs/TROUBLESHOOTING.md](../docs/TROUBLESHOOTING.md).
