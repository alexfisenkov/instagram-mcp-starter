# Метод 2 — Залогиненный браузер

Добавляет то, чего нет в официальном API: **полный архив твоего контента, скачивание постов/Reels/каруселей и сбор базы** для анализа сценариев и построения контент-плана. Работает через браузер, в котором **ты уже вошёл** в свой Instagram — агент читает и скачивает через эту сессию, ничего не публикуя.

> Только твой аккаунт, только чтение и скачивание своего. Instagram не любит автоматизацию веб-сессии — поэтому здесь запрещены любые действия (лайки, комментарии, подписки, публикации), а чтение делается спокойно, с паузами. Официальный метод 1 остаётся основным для регулярных отчётов; браузер — для архива и того, что вне API.

## Что даёт

- Полный список публикаций профиля (а не только то, что вернул API).
- **Скачивание медиа**: посты, Reels, карусели (все слайды), обложки — для локальной базы.
- Расшифровку/транскрипт видео и OCR текста на слайдах (локально) → анализ сценариев: что заходит, что нет.
- Публичные метрики, видимые в вебе, которых может не быть в текущих scopes API.

## Какой браузер подойдёт

Любой, где есть **управление агентом** и где **ты залогинен в Instagram**:

- **Встроенный браузер агента** (Claude in Chrome / браузер-панель Claude) — проще всего, если ты уже вошёл в Instagram в этом профиле.
- **Внешний Chrome/Chromium** под управлением Playwright/Chrome DevTools — если хочешь отдельный профиль под архив.

Агент сам выберет доступный ему браузерный инструмент. Главное — чтобы в нём была **живая залогиненная сессия Instagram**.

## Установка сопутствующего

Браузерное управление у агента обычно уже есть. Для **скачивания медиа** и расшифровок ставится тулинг:

```bash
# macOS / Linux — ставит gallery-dl (скачивание своих медиа) и ffmpeg (обработка/OCR-препроцессинг)
bash <(curl -fsSL https://raw.githubusercontent.com/alexfisenkov/instagram-mcp-starter/main/tools/setup-browser.sh)
```

```powershell
# Windows
irm https://raw.githubusercontent.com/alexfisenkov/instagram-mcp-starter/main/tools/setup-browser.ps1 | iex
```

Скрипт ставит (с твоего согласия): `gallery-dl` (загрузка своих постов/Reels), `ffmpeg` (кадры, аудиодорожка для расшифровки), при желании — Playwright для внешнего браузера. Транскрипт видео можно делать локально (например, через локальный Whisper), OCR текста на слайдах — локальным OCR.

## Как это делает агент (плейбук)

1. **Найти живую сессию.** Перечислить доступные браузеры/вкладки, выбрать ту, где Instagram открыт и залогинен под тобой. Если это агентский браузер — открыть instagram.com и убедиться, что нет login-wall.
2. **Не верить первому экрану.** Лента и сетка подгружаются лениво. Скроллить и ждать; не делать вывод «постов мало» по первому загруженному экрану или по стене логина.
3. **Собрать список публикаций** (ссылки, даты, тип: Reels/видео / карусель / одиночное фото). Для веб-таймлайна можно сохранять узлы GraphQL-ответов — но **не** куки, storage, токены и session-данные.
4. **Скачать медиа** через `gallery-dl` по своим ссылкам в локальную папку базы (не в облако с публичным доступом).
5. **Обработать**: для каждого элемента — транскрипт видео и OCR текста на слайдах локально, короткий разбор `analysis_*.md`, метрики в JSON.
6. **Разделить публичное и приватное.** Явно помечать источник каждой метрики: лайки/комментарии видны в вебе, а охваты, сохранения, удержание и детальная аудитория из веба берутся лишь частично даже для своего аккаунта. Глубину (удержание, отвалы) брать методом 3.
7. **Проверить количество** перед выводом «готово»: число публикаций, диапазон дат, типы, слайды каруселей, ошибки скачивания.

## Что можно и что нельзя

**Можно:** открывать свой Instagram, скроллить, читать, сохранять ссылки и публичные метрики, скачивать своё медиа, строить локальную базу.

**Нельзя (риск бана и не цель проекта):** лайкать, комментировать, подписываться/отписываться, публиковать, писать в Direct, менять профиль. Массовый парсинг чужих профилей и автодействия — вне проекта.

## Зачем это в связке

Метод 2 даёт **контент и его тексты** — сырьё для анализа сценариев. Соединяешь с цифрами: базовые из метода 1, глубина удержания — из метода 3. Вместе → понимание, какой сценарий держит зрителя, а какой теряет.
