# Метод 3 — Телефон (iOS / Android)

**Самая глубокая статистика Instagram живёт только в приложении на телефоне** и не отдаётся ни официальным API, ни вебом: удержание Reels по секундам, кривая просмотров, момент отвала аудитории, детальная аудитория Stories/Reels. Метод 3 даёт агенту контролируемый **read-only** доступ к твоему телефону, чтобы читать эти экраны: открыть приложение, дойти до нужной статистики, снять **скриншот + XML разметку экрана** (чтобы точно знать, где что находится, а не листать вслепую), забрать цифры.

> Только твой телефон и твой аккаунт. Режим по умолчанию — только чтение и навигация: открыть приложение, дойти до статистики, снять скриншот и разметку. Запрещены публикации, Direct, лайки/комментарии, follow/unfollow, изменения профиля и безопасности — если не попросишь отдельно и явно.

## Почему через разметку экрана, а не «просто скриншот»

Агент снимает не только картинку, но и **XML-дерево элементов** (accessibility source) текущего экрана. По нему он точно знает координаты и подписи кнопок/значений — можно сразу «понять, где что находится», забрать нужную метрику и перейти дальше, не пролистывая экран наугад. Скриншот — как визуальное подтверждение (evidence), XML — как «код страницы» для точной навигации.

## Вариант A — iPhone / iPad (iOS)

Требуется **Mac с Xcode** (WebDriverAgent собирается через Xcode; на Windows/Linux управлять iPhone так нельзя). Управление: **Appium + XCUITest driver + WebDriverAgent (WDA)**.

### Что нужно на телефоне

- iPhone с включённым **Режимом разработчика** (iOS 16+: Настройки → Конфиденциальность и безопасность → Режим разработчика → включить, перезагрузить).
- Подключить кабелем к Mac, на телефоне нажать **«Доверять этому компьютеру»**, оставить разблокированным.

### Установка сопутствующего (Mac)

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/alexfisenkov/instagram-mcp-starter/main/tools/setup-phone-ios.sh)
```

Скрипт (с твоего согласия) ставит: Node-инструменты Appium (`appium`, драйвер `appium-xcuitest-driver`), проверяет наличие **Xcode** и Command Line Tools, `ios-deploy`, и запускает `appium driver doctor xcuitest` — он подсказывает, чего не хватает (Developer Mode, доверие, signing). Xcode ставится из App Store вручную (большой, автоматизировать нельзя) — скрипт об этом предупредит.

### Первый запуск WebDriverAgent

WDA нужно один раз собрать под твоё устройство (после подключения нового телефона или смены версии iOS/Xcode):

1. Открой Xcode, залогинься своим Apple ID (Settings → Accounts) — бесплатного аккаунта достаточно для локальной сборки.
2. `appium driver doctor xcuitest` — пройди по его подсказкам (signing team, Developer Mode, trust).
3. Держи iPhone разблокированным и доверенным.

Типичные ошибки сборки WDA и что делать — в конце файла.

### Возможности сессии (read-only baseline)

Агент поднимает Appium-сессию с параметрами:

```
platformName = iOS
appium:automationName = XCUITest
appium:bundleId = com.burbn.instagram   (это Instagram)
appium:noReset = true                    (не сбрасывать данные/логин приложения)
appium:newCommandTimeout = 90
appium:autoAcceptAlerts = false
```

(`browserName` вместе с `bundleId` не задавать.)

Разрешено: открыть Instagram, снять screenshot и XML source, tap/swipe **для навигации к статистике**, type — только для поиска/фильтра по задаче.

## Вариант B — Android

Работает на **любом ПК** (Mac/Windows/Linux) — Xcode не нужен. Управление: **adb** (для скриншотов, дампа разметки, простых тапов) и/или **Appium + uiautomator2 driver** (для стабильной автоматизации).

### Что нужно на телефоне

- Включить **режим разработчика**: Настройки → О телефоне → 7 раз тапнуть «Номер сборки».
- Настройки → Для разработчиков → включить **Отладка по USB**.
- Подключить кабелем, на телефоне подтвердить **«Разрешить отладку по USB»** с этого компьютера.

### Установка сопутствующего

```bash
# macOS / Linux
bash <(curl -fsSL https://raw.githubusercontent.com/alexfisenkov/instagram-mcp-starter/main/tools/setup-phone-android.sh)
```

```powershell
# Windows
irm https://raw.githubusercontent.com/alexfisenkov/instagram-mcp-starter/main/tools/setup-phone-android.ps1 | iex
```

Скрипт ставит (с согласия): **platform-tools (adb)**, Appium с драйвером **uiautomator2**, по желанию **scrcpy** (зеркалирование экрана телефона на компьютер). Проверяет `adb devices` — телефон должен быть виден как `device` (не `unauthorized`).

### Быстрый read-only без Appium (только adb)

- Скриншот: `adb exec-out screencap -p > screen.png`
- Разметка экрана (тот самый «код страницы»): `adb exec-out uiautomator dump /dev/tty` → XML с элементами и их координатами.
- Открыть Instagram: `adb shell monkey -p com.instagram.android -c android.intent.category.LAUNCHER 1`
- Навигация тапом по координатам из XML: `adb shell input tap X Y`, свайп: `adb shell input swipe X1 Y1 X2 Y2 300`

Для стабильной повторяемой автоматизации — Appium + uiautomator2 (`appium:appPackage = com.instagram.android`, `appium:noReset = true`).

## Как агент достаёт глубокую статистику (плейбук)

1. **Проверить телефон.** iOS: устройство подключено и доверенное, WDA собран. Android: `adb devices` показывает `device`. Снять первый скриншот+XML — убедиться, что Instagram открыт под нужным аккаунтом.
2. **Дойти до статистики.** Профиль → «Профессиональная панель» / «Статистика» (Insights), либо конкретный Reels/пост → «Посмотреть статистику». Для Reels открыть детальную статистику удержания.
3. **На каждом значимом экране — evidence:** `screens/NN-имя.png` + `source/NN-имя.xml`. Для длинных экранов статистики — сшитый длинный скриншот. По XML извлечь числовые значения (охват, просмотры, удержание, средняя длительность просмотра, точки отвала).
4. **Для кривой удержания Reels**: открыть график удержания, снять скриншот и XML; при необходимости пройтись по ключевым точкам (начало/провалы/конец), фиксируя проценты и секунды.
5. **Свести в отчёт**: секунда → % оставшихся зрителей, где резкий отвал, средняя досматриваемость. Соединить с текстом сценария из метода 2 → вывод, какой момент теряет зрителя.

## Что НЕ сохранять (безопасность)

Никогда не выгружай в отчёты/базу и не коммить: **UDID, серийные номера, IMEI, SIM-идентификаторы** телефона; токены, куки, session-данные приложения; сырые дампы device info. В отчёт идут только скриншоты нужных экранов, XML разметки и извлечённые метрики.

## Типовые проблемы

| Симптом | Что делать |
|---------|-----------|
| iOS: `xcodebuild failed with code 70` при сборке WDA | Несовпадение версии iOS и поддержки в Xcode, или проблема подписи. Обнови Xcode/компоненты под версию iOS телефона (Xcode → Settings → Components), проверь signing team и Developer Mode, `appium driver doctor xcuitest`. |
| iOS: `Unable to find a destination matching...` | Xcode не видит устройство как цель: разблокируй, доверься, включи Developer Mode; проверь, что версия iOS поддерживается установленным Xcode. |
| Android: `adb devices` → `unauthorized` | Не подтверждена отладка на телефоне. Переподключи кабель, на телефоне нажми «Разрешить». Помогает `adb kill-server && adb start-server`. |
| Android: `device offline` | Смени кабель/порт, отключи «только зарядка» (выбери «Передача файлов»), переподключи. |
| Приложение не открывается по bundle/package | iOS bundle: `com.burbn.instagram`; Android package: `com.instagram.android`. Проверь, что приложение установлено и ты в нужном аккаунте. |
| Статистика недоступна | Инсайты в приложении есть только у профессиональных аккаунтов; у новых/маленьких аккаунтов часть метрик пуста — это ограничение Instagram. |
