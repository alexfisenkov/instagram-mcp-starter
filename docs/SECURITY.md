# Безопасность

Сервер работает локально и обращается только к официальному API Meta (`graph.facebook.com` / `graph.instagram.com` / `api.instagram.com`). Посредников нет. Но у доступа есть ключи, и обращаться с ними нужно строго.

## Три вещи, которые нельзя делать никогда

### 1. Не передавайте никому эти файлы

| Файл | Что это |
|------|---------|
| `~/.instagram-mcp/instagram.env` | App ID + **App Secret** вашего Meta-приложения |
| `~/.config/meta-instagram-mcp/token.json` | **токен доступа** к вашему Instagram |

Токен = чтение вашей статистики и комментариев от вашего имени (в пределах выданных прав). App Secret + токен = возможность продлевать доступ. Не пересылать, не класть в облако, не коммитить, не показывать на скриншотах и стримах. Оба файла создаются с правами «только владелец» — не ослабляйте.

### 2. Не вводите App Secret нигде, кроме своего instagram.env

Ни в чужие сайты, ни в чужих ботов. Единственные места, где живёт секрет: консоль Meta (App settings → Basic) и ваш локальный файл.

### 3. Пароль от Facebook/Instagram не нужен никому

Ни сервер, ни агент никогда не спрашивают ваш пароль. Пароль вы вводите только на настоящих страницах facebook.com при OAuth-входе. Если кто-то (или что-то) просит пароль в другом месте — это фишинг.

## Что видит Claude

Только то, что вы попросили прочитать: конкретные ответы API попадают в ваш диалог с Claude (обрабатываются Anthropic как любой ваш чат). `meta_auth_status` показывает токен в замаскированном виде (`EAAB...xxxx`). Сервер read-only: писать посты/комментарии от вас он не умеет.

## Экстренное отключение (мгновенно)

Любой из способов:

1. **Instagram:** Настройки → Безопасность → Приложения и сайты → удалить ваше приложение.
2. **Facebook:** Настройки → Приложения и сайты → удалить.
3. **developers.facebook.com:** App settings → удалить приложение целиком.
4. Локально: удалить `~/.config/meta-instagram-mcp/token.json` (доступ агента пропадёт, но выданное разрешение в Meta лучше тоже отозвать пунктами 1–3).

## Если ключи утекли

1. Отзовите доступ (пункты выше).
2. **App Secret можно перевыпустить:** developers.facebook.com → App settings → Basic → App secret → Reset. Старый секрет и все токены перестанут работать.
3. Пройдите OAuth заново с новым секретом.

## Для преподавателя / куратора

Для диагностики достаточно вывода доктора — он не содержит секретов. Никогда не просите у учеников `instagram.env`, `token.json` или сам App Secret.
